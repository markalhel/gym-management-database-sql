# Gym Management Database System

## Project Overview
This project is a relational database system developed to manage the daily operations of a gym. It supports the management of members, trainers, sessions, bookings, equipment, payments, attendance, and feedback.

The system demonstrates advanced SQL database design combined with Python-based analysis to simulate how data can be stored, processed, and analyzed in a real-world business environment.

This project was developed as part of the **CSPC500 SQL Databases Final Project** at the **University of Niagara Falls Canada**.

---

## Objectives
- Design a structured relational database for gym operations
- Implement SQL features such as views, stored procedures, triggers, and indexes
- Store semi-structured data using JSON and XML fields
- Generate sample operational data for analysis
- Demonstrate how Python can be used to analyze data stored in the database

## Key Features

### Member Management
Tracks gym members including contact information, subscription status, workout history, and health profiles.

### Trainer Management
Stores trainer specialties, schedules, and certifications.

### Session Scheduling
Allows trainers to create sessions and members to book them.

### Attendance Tracking
Automatically records attendance when a booking is marked as completed.

### Payment Management
Logs membership payments and transaction details.

### Equipment Monitoring
Tracks gym equipment status and maintenance history.

### Member Feedback
Allows members to rate trainers and provide feedback.
